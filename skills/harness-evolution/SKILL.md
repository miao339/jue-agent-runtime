# Harness Evolution

Use `river_harness(action="evolve_harness")`.

Append mode is for genuinely new triplets that add a new judgment direction.

Rewrite mode is for the more important cases:

- deleting bloat,
- repairing an incorrect situation,
- reordering judgment points,
- replacing a weak executable direction,
- adding a narrower ROOT fragment,
- changing the harness-specific SOUL.

When using rewrite mode, pass the complete new section fields that should replace the current ones. The tool archives the old version and writes the new `HARNESS.md` plus metadata.

After evolution, read `triplet_governance` and clean only with judgment: mark absorbed triplets `harnessed`, merge repeated ones, revoke obsolete ones, hard-delete only for mistake, corruption, or privacy.
